# Media inventory and image plan

## Inventory first

List relevant supplied and official-site assets before proposing generation. For each asset record its source, intended placement, dimensions when available, and whether ownership/reuse permission is confirmed.

Ask one direct ownership question before downloading or copying website images or logos. If permission is not confirmed, use them only to understand brand direction and create new media or placeholders.

## Plan before generation

Add one image-plan row per meaningful placement. Include:

- stable `imagePromptId`
- route and section placement
- conversion or communication purpose
- subject and composition
- aspect ratio and expected intrinsic dimensions
- source strategy
- generation/enhancement prompt

Present the complete plan with the fact/sitemap review. Generate only after approval.

## Provider order

1. Authorized supplied assets.
2. Codex image generation for a cohesive new set.
3. Magnific when a callable integration or user-controlled authenticated browser workflow is available; use for generation or enhancement as supported by the live product.
4. QLander placeholders plus `content/prompts/<slug>-image-prompts.md`.

Use one shared visual preamble and composition system across a generated set. Never generate fake testimonials, fake product interfaces presented as real, fabricated before/after evidence, unsupported badges, or text-heavy images.

## Scroll distinction

Codex and Magnific stills can support a standard page or image-led scroll narrative. Do not describe them as a continuous camera flight. `scroll-world` requires a separately approved compatible video path and its own motion, mobile, budget, and seam-QA decisions.
